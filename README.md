# logic-lab-py

> A personal 45-day journey to sharpen programming logic and problem-solving skills using Python — from fundamentals to data structures, one day at a time.

---

## What this repo is about

This is not a course. It is a structured self-study log where I work through logic building from the ground up — writing real Python code every day, committing it, and reflecting on what I learned.

The goal is simple: by the end of 45 days, be able to look at a problem, break it down, and write clean Python code to solve it without freezing up.

---

## Repo structure

```
logic-lab-py/
├── phase1_foundation/         # Days 01–10 · basics, loops, functions
├── phase2_core_patterns/      # Days 11–25 · strings, sorting, recursion, dicts
├── phase3_data_structures/    # Days 26–38 · stacks, linked lists, trees, search
├── phase4_problem_solving/    # Days 39–45 · mixed sets, refactoring, mini project
├── LEARNINGS.md               # Running log of insights and aha moments
└── README.md
```

Each day has its own `.py` file (e.g. `day01_loops.py`) with a short comment at the top describing the topic and what was practiced.

## The 45-day plan

### Phase 1 — Foundation (Days 1–10)
Build confidence with Python syntax and core control flow.

| Days | Topic |
|------|-------|
| 1–2  | Variables, data types, input/output, type casting |
| 3–4  | Conditionals — if/elif/else, nested conditions |
| 5–6  | Loops — for, while, break, continue, nested loops |
| 7    | Mini challenge — FizzBuzz, guessing game, multiplication table |
| 8–9  | Functions — parameters, return values, scope |
| 10   | Checkpoint — 5 problems using only loops + functions |

### Phase 2 — Core patterns (Days 11–25)
Learn the patterns that show up in almost every real problem.

| Days | Topic |
|------|-------|
| 11–12 | Strings — slicing, methods, palindrome, anagram |
| 13–14 | Lists — traversal, searching, 2D lists, list comprehension |
| 15–16 | Sorting from scratch — bubble sort, selection sort |
| 17    | Challenge day — duplicates, rotate array, missing number |
| 18–19 | Recursion basics — factorial, fibonacci, sum of digits |
| 20–21 | Recursion patterns — binary search, tower of hanoi |
| 22–23 | Dictionaries and sets — frequency counter, two-sum |
| 24–25 | Pattern printing — pyramids, diamonds, number patterns |

### Phase 3 — Data structures (Days 26–38)
Understand the tools behind real algorithms.

| Days | Topic |
|------|-------|
| 26–27 | Stack and queue — implement from scratch |
| 28–29 | Linked list — node class, insert, delete, reverse |
| 30–31 | Binary search — iterative and recursive |
| 32–33 | Hashing patterns — sliding window intro |
| 34–35 | Two pointers — pair sum, palindrome, remove duplicates |
| 36–37 | Binary trees — BFS and DFS traversal |
| 38    | Checkpoint — one problem per data structure |

### Phase 4 — Problem solving (Days 39–45)
Put it all together and build the habit of thinking before coding.

| Days | Topic |
|------|-------|
| 39–40 | Mixed problem sets — 3 problems/day across all topics |
| 41–42 | Optimize and refactor — revisit early solutions |
| 43–44 | Mini project — CLI tool (todo list, grade tracker, or calculator) |
| 45    | Reflection — write LEARNINGS.md, tag v1.0, plan next 45 days |

---

## Ground rules

- Minimum 45–60 minutes per day, no skipping
- Write code from scratch — no copy-pasting solutions
- Commit every day, even if it's just one working function
- When stuck, struggle for 20 minutes before looking things up
- Document what was confusing — it becomes the most useful notes
